# CLAUDE.md

## Project Overview

React Native mobile app built with Expo (managed workflow).

- **Project name**: AwsomeProject
- **Created with**: `npx create-expo-app AwsomeProject --template expo-template-blank@55`
- **React Native**: 0.83.4
- **Expo SDK**: 55
- **React**: 19.2.0

## Tech Stack

- **Framework**: React Native + Expo (managed workflow, no native /ios or /android folders committed)
- **Language**: JavaScript (no TypeScript)
- **Entry point**: `index.js` → `App.js`
- **Bundler**: Metro (via Expo)

## Commands

```bash
# Start dev server
npm start          # or: npx expo start

# Platform-specific
npm run android    # Expo Go on Android
npm run ios        # Expo Go on iOS
npm run web        # Expo web
```

## Project Structure

```
AwsomeProject/
├── App.js           # Root component
├── index.js         # Entry point (registerRootComponent)
├── app.json         # Expo configuration
├── assets/          # Icons, splash images
├── package.json
└── .vscode/         # VS Code settings
```

## Conventions

- Expo managed workflow: do not eject or modify native code directly
- Use Expo SDK-compatible libraries when adding dependencies
- App configuration is in `app.json` (Expo config)

## Notes

- This is a fresh project scaffold with no additional libraries or navigation setup yet
- The `/ios` and `/android` directories are generated by Expo and gitignored
